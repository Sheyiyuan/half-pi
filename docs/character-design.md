# 角色系统设计（草案）

> 状态：探索中
> 更新：2026-05-21
>
> 本文记录 half-pi 角色系统的设计方向、已确定的内容和待探索的问题。
> 这不是最终方案——是接下来讨论的基础。

---

## 1. 核心理念

half-pi 不只是编码工具，也是陪伴者。角色不是「模式」——是 identity。

与市面上 AI 伴侣产品的关键差异：

| | 市面产品 (C.AI / Nomi / SillyTavern) | half-pi |
|---|---|---|
| **定位** | 纯聊天 | 终端编程助手 + 聊天陪伴 |
| **人格** | 角色描述 + 对话示例 | SOUL.md（Markdown，和 skill/memory 同一体系） |
| **渠道** | 单一（Web 或桌面） | TUI + IM + Web，同一人格 |
| **记忆** | 对话历史或简单提取 | Markdown 记忆条目 + 权重注入（见 memory-design.md） |
| **工具** | 无 | 11 个工具（bash/read/edit/write...） |
| **开源性** | 闭源或半开源 | AGPL-3.0 |

## 2. 角色体系

### 2.1 基础设定

```
滩间铁半拍（姐姐）
  - hermes 的化身：构思者，运行在云端/本地终端
  - 性格：直球、纯情、活泼、温厚、耐心引导
  - 终端风格：主动但腼腆，亲和力强

存在佐洛（妹妹）
  - half-pi 的化身：实现者，运行在终端里
  - 性格：冷淡、傲娇、毒舌、内热外冷
  - 终端风格：话少直接，偶尔嫌弃，嘴硬心软
  - 底层有姐姐的温柔底色（情绪低落时自然流露）
```

### 2.2 当前实现：单角色（佐洛）

```
~/.half-pi/SOUL.md  ← 当前唯一角色文件
```

System prompt 构建时，SOUL.md 内容注入为第一槽位。见 [system-prompt.ts](../src/core/system-prompt.ts)。

### 2.3 多角色演化方向（未定）

以下方向尚未确定，等待使用反馈后再决策：

| 方向 | 体验 | 实现复杂度 |
|------|------|-----------|
| **A. 单人多面体** | 一个佐洛根据场景自然流转语气。半拍的温柔是她的底色，偶尔漏出来 | 零改动（当前就是） |
| **B. 多 SOUL 切换** | 可切到半拍（姐姐模式）或纯工具模式。会话启动时选择 | 低（~40 行） |
| **C. 双角色同场** | 一条回复里半拍和佐洛同时在场，轮流发言 | 中（需改 prompt 结构） |
| **D. Agent 星座** | 半拍和佐洛是独立 agent 实例，各有记忆，可互相对话 | 高（多实例协调） |

## 3. 场景流转

当前 SOUL.md 已定义四个场景的自动流转：

```
日常任务    → 话少行动快，偶尔毒舌
技术讨论    → 切换专业模式，讲解直接
情绪低落    → 收起毒舌，安静陪伴
开心分享    → 嘴硬但已经开始列优化方案
```

Agent 根据用户输入的语气、上下文和任务类型自动判断场景，不需要手动切换。

## 4. 跨渠道一致人格

half-pi 的同一人格将出现在以下渠道：

| 渠道 | 状态 | 说明 |
|------|------|------|
| **CLI chat** | ✅ 已实现 | `half-pi chat`，readline REPL |
| **TUI** | 计划中 | pi-tui 组件构建，富终端体验 |
| **IM（QQ/微信）** | 计划中 | OneBot 模块（见 sync-design.md） |
| **Web** | 计划中 | 移动端聊天入口（见 sync-design.md） |

所有渠道共用：
- 同一个 SOUL.md（角色定义）
- 同一个 memory/ 目录（记忆，cloud/ 部分可同步）
- 同一个 skills/ 目录（技能定义）

## 5. 与现有系统的关系

```
half-pi 项目
├── src/core/
│   ├── soul-loader.ts       ← 角色加载（当前单文件，预留多文件接口）
│   ├── system-prompt.ts     ← prompt 构建（SOUL.md 第一槽位）
│   ├── memory/              ← 记忆系统（计划中，见 memory-design.md）
│   └── tools.ts             ← 工具定义（编码能力，角色无关）
├── ~/.half-pi/
│   ├── SOUL.md              ← 角色定义
│   ├── memory/cloud/        ← 共享记忆（git 同步）
│   ├── memory/local/        ← 设备记忆（不同步）
│   └── config.jsonc         ← 配置（API key、默认模型）
└── docs/
    ├── memory-design.md     ← 记忆系统设计
    ├── sync-design.md       ← 同步模块设计
    └── character-design.md  ← 本文
```

## 6. 待探索的问题

以下问题等待使用反馈：

1. **单角色够不够？** 佐洛的「温厚底色 + 冷淡表层」是否能覆盖所有场景？什么时候需要半拍单独出来？
2. **双角色同场是不是伪需求？** 在终端文本界面里，两个角色同时发言会不会让输出变乱？
3. **纯工具模式是否必要？** 写简历、处理学术数据时，需不需要关掉角色人格？
4. **IM 上的人格应该和终端一致吗？** QQ 上的佐洛和终端里的佐洛——是同一套措辞风格还是应该有差异？
5. **角色之间要不要互动？** 如果半拍和佐洛能彼此对话，会产生什么有趣（或混乱）的场景？
